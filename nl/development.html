Ontwikkeling

We zijn in iedere bijdrage geïnteresseerd. Er zijn veel manieren om een bijdrage te leveren:

  • Als je iets interessants gemaakt hebt met SymPy, stuur ons dan graag een patch op met de code (zie hieronder voor meer informatie).
  • Als je een bug vindt (of je wilt over bepaalde issues je gedachten delen), laat het ons weten via de mailinglist. Je kunt ook een bug rapport (of opmerking) toevoegen aan de issues zelf.
  • Je kunt bloggen over hoe je SymPy gebruikt.
  • Je kunt ons helpen met het schrijven van documentatie (door ons gewoon een patch toe te sturen).
  • Je kunt mensen op onze mailinglist helpen.
  • Je kunt ons helpen met het beoordelen van patches.
  • Je kunt ons patches toesturen met bugfixes of nieuwe programmaonderdelen.

De beste manier om code of documentatie bij te dragen, is door ons een patch op te sturen.

Hoe je een Patch opstuurt

De beste manier om een Patch op te sturen, is via een GitHub Pull Requestvoor de sympy/sympy repository.

Als je hulp nodig hebt met git, github, pull requests, patches, code schrijven, Sympy of andere gerelateerde zaken, stuur ons dan een email via de mailinglist of vraag het ons op IRC (ons kanaal is #sympy at freenode), we zullen helpen. Je kunt ook onze instructies Hoe bereid je een patch voor met git and github lezen.

Onze voorkeur gaat uit naar de GitHub Pull Request methode, dit maakt het voor ons makkelijk om je code te bekijken en de code toe te voegen. Je kunt de meest recente git repository clonen (zie de link rechts), een branch creëren met jouw code en deze ergens uploaden (naar github bijvoorbeeld).Vervolgens kun je ons een link sturen naar de sympy-patches mailinglist, of ons de raw patches sturen -- dit is echter meer werk voor ons wat betreft de integratie.

SymPy heeft een breed team aan ontwikkelaars: http://docs.sympy.org/0.7.1/aboutus.html#sympy-development-team, stuur ons een patch en je naam zal daar ook genoemd worden.
