Entwicklung

Wir sind an jeder Beteiligung interessiert. Es gibt viele Möglichkeiten, etwas zu SymPy beizutragen:

  • Wenn Du Interessantes mit Hilfe von SymPy umgesetzt hast, sende uns bitte einen Patch mit dem Code (weitere Informationen siehe unten).
  • Wenn Du einen Fehler gefunden hast (oder uns einfach sagen möchtest, was Du denkst), erzähle es uns in unserem Mailverteiler. Außerdem kannst du unsere Fehlerliste vervollständigen.
  • Du kannst darüber bloggen, wie man SymPy benutzt.
  • Du kannst uns helfen, die Dokumentation zu schreiben (sende uns einfach einen Patch).
  • Du kannst Leuten in unserem Mailverteiler helfen.
  • Du kannst uns helfen, die Patches zu begutachten.
  • Du kannst uns Patches mit Fehlerkorrekturen oder neuen Features senden.

Am effektivsten hilfst du, indem du uns einen Patch schickst.

Einsenden von Patches

Am besten ist es, einen Pull-Request auf Github an das sympy/sympy Repository zu senden. Wir werden ihn dann begutachten und einfügen.

Falls Du Hilfe mit git, github, pull requests, Patches, Programmieren, SymPy oder verwandten Themen brauchst, sende uns einfach eine Email an unsere Mailing List oder frage in unserem IRC-Kanal (#sympy auf freenode), wir werden dir helfen. Außerdem kannst Du unsere Anleitung Wie man Patches mit git und github vorbereitet lesen.

Pull-Requests auf Github sind die bevorzugte Methode, da es für uns so am einfachsten ist, den Code zu begutachten und zu pushen. Abgesehen davon kannst Du auch das aktuellste git-Repository klonen (siehe Link rechts), einen neuen Zweig mit deinem Code vorbereiten und irgendwo hochladen (zum Beispiel github), und uns einen Link an den sympy-patches Email-Verteiler, oder auch nur den Code selbst senden --- letzteres ist jedoch aufwändiger zu integrieren.

SymPy hat ein großes Entwicklungsteam: http://docs.sympy.org/0.7.1/aboutus.html#sympy-development-team . Schick' uns einen Patch, und auch dein Name erscheint dort.
