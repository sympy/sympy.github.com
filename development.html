Entwicklung

Wir sind an jeder Beteiligung interessiert. Es gibt viele Wege, etwas zu SymPy beizutragen:

  • Wenn du Interessantes mit Hilfe von SymPy umgesetzt hast, sende uns bitte einen Patch mit dem Code (siehe unten für weitere Informationen).
  • Wenn du einen Bug gefunden hast (oder uns einfach sagen möchtest was du denkst), erzähle es uns in unserer Mailing List. Du kannst auch einen Bug Report (oder ein Kommentar) zu unserem Issues Tracker hinzufügen.
  • Du kannst darüber bloggen wie man SymPy benutzt.
  • Du kannst uns helfen die Dokumentation zu schreiben (sende uns einfach einen Patch).
  • Du kannst Leuten in unserer Mailing List helfen.
  • Du kannst uns helfen die Patches zu überprüfen.
  • Du kannst uns Patches mit Bugfixes oder neuen Features senden.

Die beste Möglichkeit mitzuhelfen besteht darin, uns einen Patch zu senden.

Wie man einen Patch sendet

Am besten ist es, einen github pull request an die sympy/sympy Repository zu senden. Wir werden ihn dann begutachten und einfügen.

Falls du Hilfe mit git, github, pull requests, Patches, coding, SymPy oder verwandten Themen brauchst, sende uns einfach eine eMail an unsere Mailing List oder frage in unserem IRC Channel (#sympy auf freenode), wir werden dir helfen. Außerdem kannst du unsere Anleitung Wie man Patches mit git und github vorbereitet lesen.

github pull requests sind die bevorzugte Methode, da es für uns so am einfachsten ist, den Code aufzuarbeiten und zu pushen. Abgesehen davon, kannst du auch die aktuellste git Repository klonen (siehe Link rechts), einen neuen Zweig für deinen Code vorbereiten und anderswo hochladen (zum Beispiel github), und uns einen Link an die sympy-patches Mailing List, oder auch nur den Code selbst senden --- letzteres ist jedoch aufwändiger zu integrieren.

SymPy hat ein großes Entwicklungsteam: http://docs.sympy.org/0.7.1/aboutus.html#sympy-development-team . Sende uns einfach einen Patch und dein Name erscheint auch dort.
