Développement

Nous sommes intéressés par toutes formes de contribution. Vous pouvez contribuer de diverses façons.:

  • Si vous avez écrit quelque chose d'intéressant utilisant Sympy, n'hésitez pas à nous soumettre un patch avec le code (pour plus d'informations, voir ci-après).
  • Si vous avez trouvé un bug (ou si vous souhaitez nous faire part de vos remarques), dites-le nous sur la liste de diffusion. Vous pouvez aussi rédiger un rapport de bug ou un commentaire dans le système de suivi de problèmes.
  • Vous pouvez blogguer à propos de la façon dont vous utilisez Sympy.
  • Vous pouvez nous aider à écrire la documentation (envoyez-nous juste un patch).
  • Vous pouvez venir en aide à des utilisateurs sur notre liste de diffusion.
  • Vous pouvez aider en passant en revue des patches.
  • Vous pouvez nous envoyer des patches pour corriger des bugs ou ajouter de nouvelles fonctionnalités..

La meilleure façon de contribuer au code ou à la documentation est de nous envoyer un patch..

Envoyer un patch

La meilleure solution est d'envoyer une requête de pull sur le dépôt github sympy/sympy. Nous le passerons en revue, puis nous l'intègrerons.

Si vous avez besoin d'aide concernant git, github, les requêtes de pull, les patches, le code, SymPy ou quoi que ce soit d'autre, envoyez-nous un mail via la liste de diffusion ou posez vos questions sur IRC (notre canal est #sympy at freenode) et nous vous aiderons. Vous pouvez aussi lire nos instructions sur comment preparer des patches à l'aide de git et github.

La requête de pull sur github est la méthode privilégiée, car elle nous permet de passer en revue et d'intégrer le code plus aisément. Ceci dit, vous pouvez aussi cloner la dernière version du dépôt git (référez-vous au lien à droite), préparer une branche avec votre code, héberger votre code quelque part (par exemple sur github), puis nous envoyer un lien vers la liste de diffusion sympy-patches. Vous pouvez aussi nous envoyer des patches bruts, mais cela demandera plus de travail pour les intégrer.

SymPy dispose d'une grande équipe de développement: http://docs.sympy.org/0.7.1/aboutus.html#sympy-development-team , envoyez-nous un patch et votre nom apparaîtra ici.
