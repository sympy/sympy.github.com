Desarrollo

Estamos interesados en cualquier contribución. Hay muchas maneras de contribuir:

  • Si escribiste algo interesante usando SymPy, por favor envíanos un parche con el código (mira abajo para más información).
  • Si encontraste un bug (o solo quieres darnos tu opinión), dilo en la lista de correos. También puedes agregar un reporte del bug (o un comentario) dentro de issues.
  • Crea tu blog acerca de como usar SympPy .
  • Puedes ayudarnos a escribir documentación (solo envia un parche).
  • Puedes ayudar en nuestra lista de correos.
  • Puedes ayudar revisando parches.
  • Puedes enviarnos parches para bugs o nuevas funcionalidades.

La mejor manera de contribuir con código o documentación es enviando un parche.

Cómo Enviar un Parche

La mejor manera es enviando un pull request a nuestro github: sympy/sympyNosotros lo revisaremos y le haremos un push

Si necesitas ayuda con git, github, pull requests, parches, código, SymPy o algo relacionado, solo envia un mail a nuestra lista de correos o pregunta en IRC(nuestro canal es #sympy en freenode), Nosotros te ayudaremos. También, puedes leer nuestra instrucciones Cómo preparar parches usando git y github.

Pull request en github es el método preferido, ya que se nos hace más fácil revisar y hacer push al código. También puedes clonar nuestro repositorio en git (mira el link a la derecha), preparar un branch con tu código, súbelo en algun lugar (por ejemplo github) y envíanos el link a sympy-patches nuestra lista de correos, o también podrias enviarnos parches brutos --- pero sería más trabajoso para poder integrarlo.

SymPy tiene un gran equipo de desarrollo: http://docs.sympy.org/0.7.1/aboutus.html#sympy-development-team , envíanos un parche y tu nombre aparecerá allí también.
