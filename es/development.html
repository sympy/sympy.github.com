Desarrollo

Estamos interesados en cualquier contribución. Hay muchas maneras de contribuir:

  • Si escribiste algo interesante usando SymPy, por favor envíanos un parche con el código (mira abajo para más información).
  • Si encontraste un bug (o solo quieres darnos tu opinión), dilo en la lista de correos. You can also add a bug report (or a comment) into the issues.
  • Crea tu blog acerca de como usar SympPy .
  • Puedes ayudarnos a escribir documentación (solo envia un parche).
  • Puedes ayudar en nuestra lista de correos.
  • Puedes ayudar revisando parches.
  • Puedes enviarnos parches para bugs o nuevas funcionalidades.

La mejor manera de contribuir con código o documentación es enviando un parche.

Cómo Enviar un Parche

La mejor manera es enviando un pull request a nuestro github: sympy/sympyNosotros lo revisaremos y le haremos un push

If you need help with git, github, pull requests, patches, coding, SymPy or anything related, just send us an email to the mailinglist or ask on Gitter, we'll be happy to help. También, puedes leer nuestra instrucciones Cómo preparar parches usando git y github.

The github pull request is a preferred method, as that makes it easy for us to review and push the code in. That said, you can also clone the latest git repository (see the link on the right), prepare a branch with your code, upload it somewhere (for example github) and send us a link to the sympy mailinglist, or you can even send us raw patches --- but it will be more work for us to integrate it.

SymPy has a broad development team, send us a patch and your name will appear there too.
